# Social archive

Snapshots of posts and threads (for example **X.com**) related to **VincentAtHome** live under `threads/` as Markdown so they stay in the repo without depending on a live login.

## Add a thread

1. Open the thread on X.
2. Note the canonical URL (any post in the thread works; it looks like `https://x.com/<user>/status/<id>`).
3. Create `threads/<short-slug>.md` (or ask in chat to have it generated) with:
   - **Source:** the URL  
   - **Captured:** date (UTC)  
   - **Posts:** copy of each tweet in order (author, text, time if you like)

X often blocks unauthenticated scraping, so **pasting the thread text** into a file is the most reliable archive.
